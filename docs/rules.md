# rules

## architecture/

**“Why is ___ designed this way?”**

- Design and intent
- No step-by-step commands    
- Why choices were made
- Diagrams 

## runbooks/

**“How do I perform ___ safely?”**

- Repeatable procedures
- Step-by-step commands
- Workflows
    

## troubleshooting/

**“Why did this break and how did I fix it?”**

- Problem > root cause > fix
- Error-message driven
- Searchable
    

## cheat-sheets/

**Quick CLI recall**

- Dense command reference
- No narrative
- No long explanations

## inventory/

 **“What exists right now?”**

- Source-of-truth tables
- Structured information
- Snapshot of current state

## security/

**“How is this protected?”**

- Hardening posture
- Access model
- Key/cert handling
